Lecture 2

What is Computer and its Applications

1

What is Computer?

  • Derived from the word compute, which means to calculate.
  • A computer is an electronic machine that:
    • Accepts data from the user,
    • Processes data by performing calculations and operations,
    • Generates the desired output results.
  • Performs simple and complex operations with speed and accuracy.
2

Data & Information

  • Data: Raw, unorganized facts with no meaning.
  • Information: Processed data that has meaning.
3

Input-Process-Output Concept

Computer accepts input → Processes data → Generates output → Stores data. This is called the input-process-output concept.

4

Input

  • Data accepted via input devices like keyboard.
  • Data types: text, sound, images, documents.
  • Processing actions: arithmetic/logic calculations, editing, modifying documents.
  • Stored temporarily in main memory during processing.
5

Storage

  • Permanent storage in secondary devices (disk, tape).
  • Stored data can be retrieved later.
6

Output

  • Result after processing.
  • Forms: text, sound, image, document.
  • Output methods: monitor display, printer, sound playback.
7

Characteristics of Computer

  • Speed: Faster than humans in computation.
  • Accuracy: Programmed for error-free results.
  • Storage: Large data storage.
  • Diligence: Works long hours without error.
  • Versatility: Can perform different tasks.
  • No IQ: Needs instructions to work.
  • No Feeling: Lacks emotions, experience, or knowledge.
8

Uses of Computer

  • Integral to life: at home, office, movies, hotels, driving.
  • Manages storage, flow, processing of data.
9

Applications of Computer

  • Education: Notes, presentations, e-learning, online exams.
  • Government: E-governance, online services, crime databases.
  • Sports: Score tracking, game development, training analysis.
  • Advertising: Digital ads, email marketing, web ads, visual & sound effects.
  • Medicine: Research data, medical history, diagnostic equipment (ultrasound, MRI).
  • Science & Engineering: Scientific calculations, CAD/CAM, space research.
  • Home: Games, accounts, communication, bill payments, embedded microprocessors.
10

Data vs Information Comparison

Basis Data Information
Meaning Raw facts about someone or something Refined facts concerning a specific subject
Form Text and numbers Organized
Useful May or may not be useful Always useful
Dependency Does not depend on information Cannot exist without data
Lecture 3

The Computer Generations

1

Overview

“Generations” distinguish technological steps in computer evolution, now including hardware and software.

2

First Generation (1946–1959): Vacuum Tubes

  • Used vacuum tubes for circuitry and magnetic drums for memory.
  • Input: Punched cards and paper tapes; Output: Printouts.
  • Instructions in machine language (binary).
  • Solved one problem at a time.
  • Large size, required large room.
  • Examples: UNIVAC, ENIAC, EDVAC
3

Second Generation (1959–1965): Transistors

  • Transistors replaced vacuum tubes → smaller, faster, cheaper, energy efficient.
  • Magnetic tapes/disks for storage.
  • Input: Punched cards; Output: Printouts.
  • Used assembly language (mnemonics like ADD, SUB).
  • High-level languages introduced: FORTRAN, COBOL, ALGOL, SNOBOL.
  • Examples: PDP-8, IBM 1401, CDC 1604.
  • Used for business applications: payroll, inventory control, etc.
4

Third Generation (1965–1971): Integrated Circuits (IC)

  • IC chips combined multiple components on a silicon chip.
  • Small Scale Integration (SSI) → Medium Scale Integration (MSI).
  • Interaction via keyboard and monitor (no more punched cards).
  • High-level languages extended: PL/1, PASCAL, BASIC.
  • Examples: IBM 370, PDP 11.
5

Fourth Generation (1971–1980): Microprocessors

  • Large Scale Integration (LSI) & Very Large Scale Integration (VLSI).
  • Microprocessor: Single chip performing arithmetic, logic, control.
  • Led to Personal Computer (PC) revolution.
  • Semiconductor memories replaced magnetic cores.
  • Networking and the rise of the Internet.
  • Operating Systems: MS-DOS, MS-Windows, UNIX.
  • GUI introduced for user-friendly interaction.
6

Fifth Generation (1980 Onwards): Artificial Intelligence

  • Goal: Computers capable of learning and self-organization.
  • Ultra Large Scale Integration (ULSI): Millions of components on a chip.
  • Parallel processing: Multiple instructions at once → faster speed.
  • Based on Artificial Intelligence (AI), simulating human thinking and reasoning.
7

Generation Summary

Generation Period Technology Used
First 1946–1959 Vacuum Tubes
Second 1959–1965 Transistors
Third 1965–1971 Integrated Circuits (IC)
Fourth 1971–1980 Microprocessors (LSI, VLSI)
Fifth 1980–Present ULSI, AI-based

This table provides a quick revision of all computer generations.

General Purpose Computers

Used for general tasks like sales analysis, financial accounting, invoicing, etc.

Designed for specific tasks requiring dedicated computation.

  • Applications: Weather forecasting, space applications, medical diagnostics

According to technology used

Analog Computers

Store data as continuously varying physical quantities (voltage, current).

Digital Computers

Hybrid Computers

According to size and capacity

Microcomputers

Small, low‑cost, single‑user systems.

  • Examples: Desktop PC, Laptop, Smartphone, Tablet

Minicomputers

Multi‑user systems supporting about 4–200 users.

  • Use cases: Industries, research centers
  • Examples: PDP‑11, IBM 8000 series

Mainframe Computers

High performance with large storage for many users.

  • Use cases: Banks, large organizations, centralized databases
  • Examples: CDC 6600, IBM ES000 series

Supercomputers

Fastest and most expensive; for highly calculation‑intensive tasks.

  • Metric: Performance measured in FLOPS
  • Examples: IBM Roadrunner, PARAM Padma (India, 1 TFLOP), AIRAWAT (India, 13,170 TFLOPs), Tainhe‑2 (China), Titan, Frontier (America)

Components of a Digital Computer

1

Components of a Digital Computer

  • Characteristics of Computer:
    1. Responds to a specific set of instructions.
    2. Executes pre-recorded instructions (a program).
  • Hardware: Physical components (wires, circuits).
  • Software: Instructions and data.
2

Basic Units of Computer System

  • Central Processing Unit (CPU):
    • Executes program instructions.
    • Consists of:
      • Control Unit (CU): Directs electronic signals and controls data flow between memory, ALU, and I/O devices.
      • Arithmetic Logic Unit (ALU): Performs arithmetic (add, subtract, multiply, divide) and logical comparisons.
  • Storage Unit:
    • Holds instructions, data, intermediate, and final results.
    • Memory is organized in bytes (1 byte = 8 bits).
3

Types of Storage Units

  • Primary Memory:
    • Holds running programs and data.
    • Fast, small capacity, volatile, expensive.
    • Types:
      • RAM (Random Access Memory):
        • Volatile, fast, expensive.
        • SRAM: Faster, no refresh needed, used as cache memory.
        • DRAM: Needs refresh, predominant in modern computers.
      • ROM (Read Only Memory):
        • Non-volatile, stores instructions for bootstrapping.
        • Types: PROM, EPROM, EEPROM.
      • CMOS Memory: Stores system settings like date, time, boot devices.
  • Secondary Storage:
    • Large capacity, slower than primary memory, non-volatile, cheaper.
    • Examples: HDD, CD-ROM, DVD, Pen Drive, Memory Card.
4

Input Unit

  • Reads instructions and data from outside.
  • Converts to computer-acceptable form.
  • Examples: Keyboard, Mouse, Scanner.
5

Output Unit

  • Converts coded results to human-readable form.
  • Examples: Monitor, Printer, Speaker.

Lecture 10/11

Problem Solving Concepts

Steps • Design • Coding • Debugging • Documentation • Maintenance

Overview

This lecture walks through how to move from a problem to a reliable software solution: analysis, design, coding, testing, documentation, and maintenance.

Contents

  • 1. Concept of problem solving
  • 2. Steps of problem solving
  • 3. Problem definition/specification
  • 4. Program design
①

Concept of problem solving

Core steps involved in solving a computer problem:

  1. Understanding and describing the problem clearly.
  2. Designing an algorithm.
  3. Developing a computer solution.
  4. Testing the solution.
  5. Documentation.
  6. Maintenance.
②

Steps of problem solving

Program analysis

  • Goal: Analyze the problem and desired output.
  • Requirements: Gather and clarify constraints.
  • Scope: Define what the program should do.
  • Process: Define processing tasks clearly.

Program design

Design the solution without writing code.

  • Tools: Flowchart, Algorithm, Pseudocode.
  • Outcome: A design that can be translated into code.
  • Validation: Output must solve the intended problem.

Program coding

  • Action: Write the actual code using a programming language.
  • Practice: Follow standards, name clearly, keep modules small.

Program debugging

  • Definition: Finding and fixing errors (“bugs”).
  • Testing: Ensure all modules work correctly via thorough tests.

Program documentation

  • Purpose: Help users and future developers.
  • Includes: Problem Definition, Design, Test Docs, Development History, User Manual (input, processing, output).

Program maintenance

  • Corrections: Detect and eliminate undetected errors.
  • Improvements: Modify to enhance performance and adaptability.
  • Evolution: Add new features as needs change.
③

Problem definition/specification

Precise definitions produce better solutions. Always specify inputs and outputs.

Example problem

  • Task: Compute the average of three numbers.
  • Clarify: What is “average”? How is it calculated?

Specification template

  • INPUT: Three numeric values, say a, b, c.
  • OUTPUT: Average value (a + b + c) / 3.

Sample algorithm (pseudocode)

  • Step 1: Read a, b, c.
  • Step 2: avg ← (a + b + c) / 3.
  • Step 3: Print avg.
④

Program design

Before coding, follow these steps to model and verify your solution.

  1. Understand thoroughly: Restate the problem; list assumptions and constraints.
  2. Model the solution: Use flowcharts, algorithms, and pseudocode to map logic.
  3. Create test data: Cover normal, edge, and error cases to verify correctness later.

Tip

Keep designs simple and modular. Each module should do one thing well and be easy to test.

Lecture 12

Debugging and Types of Errors

1

Debugging

Debugging is the process of finding and fixing errors or bugs in the source code of any software. When software does not work as expected, programmers study the code to determine why and where the errors occurred.

2

Types of Errors in a Program

  • Syntax Error
    • Syntax refers to the structure and rules of the program.
    • Occurs when the interpreter cannot understand a line of code (e.g., missing parentheses).
    Example: 
    print "Hello World!"

    Output: Syntax error because parentheses are missing.

  • Logical Error
    • Produces unpredictable results or incorrect outputs.
    • Hard to detect.
    Example: 
    x = 3
y = 4
average = x + y / 2
print(average)

    Output: 5.0 (incorrect; should be (x + y) / 2 = 3.5).

  • Runtime Error
    • Occurs while the program executes.
    • Python understands the code but cannot perform the action.
    Example 1 — Undefined variable: 
    day = "Sunday"
print(Day)

    Output:

    NameError: name 'Day' is not defined
    Example 2 — Division by zero: 
    no = 0
num = 2 / no

    Output:

    ZeroDivisionError: integer division or modulo by zero
Lecture 13/14

Techniques of Problem Solving

1️⃣

Purpose of Program Planning

  • Correct program likhne ke liye, har instruction ko sahi sequence me likhna jaruri hai.
  • Program planning se ensure hota hai ki instructions:
    • ➔ Problem ke liye appropriate ho
    • ➔ Sahi sequence me ho
2️⃣

Algorithm

  • Step-by-step process to solve any logical and mathematical problem.
  • Definition (Thomas H. Coreman): ➔ Sequence of computational steps transforming input to desired output.
3️⃣

Characteristics of Algorithm

  • Input: At least one input.
  • Output: Desired output.
  • Definiteness: Clear intent.
  • Finiteness: Finite steps.
  • Effectiveness: Achieves intended purpose.
4️⃣

Flowchart

Diagrammatic representation of problem-solving process.

Symbols

  • Oval → Start/Stop
  • Rectangle → Process
  • Parallelogram → Input/Output
  • Diamond → Decision
  • Arrow → Flow of steps

Levels

  • Macro flowchart → Main segments only
  • Micro flowchart → Detailed steps

Rules to Design

  • One Start, one Stop
  • Top to bottom, left to right flow
  • Simple, clear words
  • No crossing arrows
5️⃣

Advantages of Flowchart

  • Better communication
  • Proper documentation
  • Efficient coding
  • Systematic debugging & testing
6️⃣

Limitations of Flowchart

  • Time-consuming to draw
  • Redrawing for changes is tedious
  • No standard detail level
7️⃣

Example Algorithms

Sum of Two Numbers

  1. Start
  2. Input A, B
  3. SUM = A + B
  4. Display SUM
  5. Stop

Area and Perimeter of Square

  1. Start
  2. Input side L
  3. Area = L × L
  4. Perimeter = 4 × L
  5. Display Area, Perimeter
  6. Stop

Cube of a Number

  1. Start
  2. Input number A
  3. Cube = A × A × A
  4. Display Cube
  5. Stop
8️⃣

Pseudocode

Natural language resembling computer instructions. Easier than flowcharts to modify.

Example — Adding Two Numbers

BEGIN
NUMBER s1, s2, sum
INPUT s1
INPUT s2
sum = s1 + s2
OUTPUT sum
END
